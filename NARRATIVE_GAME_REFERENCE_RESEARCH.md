# Narrative Game Reference Research

本文件用于回答：这个在华留学体验游戏如果采用视觉小说式游玩方式，为什么仍然要“像游戏”，以及接下来应该补哪些可玩层。

注意：本项目不是恋爱游戏，也不应以恋爱为核心卖点。可借鉴的是视觉小说、叙事 RPG、生活模拟叙事游戏的节奏、反馈和回看机制。

## 1. 参考对象

### 80 Days

来源：Inkle 官方页面  
https://www.inklestudios.com/80days/

可借鉴点：

- 它把文字选择和路线规划绑定在一起，玩家不是只读故事，而是在时间、路线、风险、资源之间做取舍。
- 官方强调高重玩性、每次旅程独特、数千个选择和 150 个城市，这说明“同一主题下的大量局部变化”会显著提升探索欲。
- 对本项目的启发：留学流程可以像一次“从申请到抵达再到未来”的路线旅行。Calendar、城市移动、每周目标、结局条件都应该让玩家感觉自己在规划一段人生路线，而不是只点击下一段文字。

落地建议：

- Semester Calendar 继续强化为路线规划工具，而不是普通日志。
- 每个阶段给玩家 2-3 个明确 deadline：申请截止、签证预约、注册窗口、midterm、future direction、year-end review。
- 上海城市事件不要只是旅游点，应该和资源、人物、机会挂钩，例如 Lujiazui 对 career/digital，Fuxing Park 对 culture/local，Bund 对 reflection/ending memory。

### Citizen Sleeper

来源：Fellow Traveller 官方页面  
https://www.fellowtraveller.games/citizen-sleeper

可借鉴点：

- 官方明确写到它使用 dice、clocks、drives 形成玩家主导体验。
- 每个 cycle 玩家分配行动，角色、阵营、资源和未来方向相互拉扯。
- 好玩的核心不是操作复杂，而是“每天/每轮只有有限行动”，每个选择都感觉在消耗真实机会。

落地建议：

- 本项目不需要上复杂骰子，但可以保留“每周只能做有限事情”的压力。
- SimPad 应该显示短期 drive：本周最该处理的 deadline、一个关系机会、一个经济机会、一个恢复机会。
- 关系角色不只是好感条，应该成为 drive 入口：WeChat meetup、office hours、demo day、festival prep 等。

### Coffee Talk

来源：Toge Productions 官方 itch 页面  
https://togeproductions.itch.io/coffee-talk

可借鉴点：

- 官方描述它是通过聆听问题、用手头材料做饮品来帮助角色。
- 分支不一定来自传统对白选项，而来自玩家如何服务、如何回应角色需求。
- 它的低强度操作让玩家有“我在参与这个场景”的感觉，而不是旁观。

落地建议：

- 本项目可把 Taobao、DiDi、WeChat 做成“服务式选择”：不是按钮加数值，而是解决一个生活问题。
- 例如 Taobao 的 wrong-address recovery、meal prep、dorm fix，本质上就是留学生活里的“调饮品”：玩家用工具解决具体摩擦。
- 角色支线可以更生活化：不是每次都重大抉择，而是帮 Sophie 整理 guide、帮 Neighbor Li 搬节日物资、帮 Xiao Chen 看投诉截图。

### VA-11 HALL-A

来源：官方页面  
https://waifubartending.com/

可借鉴点：

- 官方说明它的分支并不依赖传统选择，而是通过玩家调制的饮品影响故事。
- 核心乐趣是固定职业场景里的重复操作、角色来访、口味记忆和故事揭露。
- 它证明“轻操作 + 强角色文本 + 记忆回访”足以支撑视觉小说式体验。

落地建议：

- 本项目可以把 SimPad 作为“留学生的吧台”：角色、城市、账单、快递、交通都从手机里来。
- 玩家应该逐渐记住角色偏好：Professor Lin 要可靠草稿，Manager Zhang 要合法准备，Xiao Chen 要真实用户，Sophie 要可复制的支持系统。
- Gallery / Memory Archive 很重要，因为它把“我经历过”变成可回看的奖励。

### Overboard!

来源：Inkle 官方页面  
https://www.inklestudios.com/overboard/

可借鉴点：

- 官方强调角色会记住玩家看见、听见、做过的事情。
- 它把时间限制、角色独立行动、线索和重玩性放在一起。
- 对本项目最有价值的是“记忆系统”：玩家之前怎么处理人、钱、规则，后面应该被剧情记住。

落地建议：

- 继续把 flags 外显到 SimPad：Route Memory、Relationship Tension、Risk Status 是正确方向。
- 后续应让更多中期选择回流到结局 afterword，而不是只影响数值。
- 可以加入“角色记得你如何做事”的小反馈，比如 Professor Lin 后续引用你曾经修过的 draft，Manager Zhang 记得你有没有走 shortcut。

## 2. 对本项目的综合结论

目前项目最大问题不是剧情量不够，而是“玩家动作之间的区分度还不够”。玩家会觉得一直点击，是因为很多选择虽然有数值反馈，但没有足够多的即时界面变化、角色回访和可回看奖励。

要解决这个问题，不需要推翻现有结构。现在最应该增强的是：

- 每周转换要被看见：Week transition screen / Next Week button 已经是必要方向。
- SimPad 要从信息面板变成生活工具：Calendar、Taobao、DiDi、WeChat meetup、Gallery 都要继续实装。
- 角色要成为玩法入口：不是只在剧情里出现，而是在 WeChat 中主动触发支线。
- CG 要成为奖励系统：关键事件、角色 commitment、ending 都应该进入 Memory Archive。
- 数值反馈要更具体：不要只显示 `+Culture`，最好让玩家知道“为什么这次选择提升了文化理解/数字能力/关系网络”。

## 3. 已经开始落地的改动

当前已落地：

- SimPad 新增 Calendar tab，显示阶段 deadline 和本周关注点。
- SimPad 新增 Gallery / Memory Archive，能回看已解锁 CG。
- WeChat 联系人中新增可触发的 meetup 入口，用角色关系解锁支线节点。
- DiDi 新增 City Shortcuts，用交通 app 直接启动上海场景。
- Taobao 新增 Service Orders，用每周订单制造生活工具感。

这些改动对应上面的参考结论：它们都在减少“纯点文本”，把留学生活拆成可管理、可触发、可回看的小系统。

## 4. 下一阶段建议任务

优先级 P0：

- 做实际试玩 QA，确认 Calendar、Gallery、DiDi Shortcuts、Taobao Service Orders、WeChat meetup 都能在游戏内正常显示与触发。
- 给 Week transition 加更明显的过渡界面或 Next Week 按钮，避免玩家错过周变化。
- 扩展 Memory Archive 的解锁判断，让 ending flags 真实写入，避免玩家通关后 CG 不进图库。

优先级 P1：

- 给每条主路线补 1 个更强的“角色面对你说话”的中后期 CG。
- 给 SimPad 的 Calendar 增加“本周推荐行动”，例如如果 Energy 太低推荐恢复，如果 money 低推荐 budget review。
- 把 Taobao / DiDi 的每周操作做成小事件池，而不是固定三项。

优先级 P2：

- 做角色偏好系统：每个角色偏好不同类型选择，后续对话记得玩家的做法。
- 做 Replay / Memory filter：按 Application、Pre-Departure、Academic、Career、Local、International、Shanghai、Risk 分类查看。
- 做更多低强度互动小游戏，但每个小游戏都必须服务剧情，不要为操作而操作。
