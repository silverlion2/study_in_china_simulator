# 前期开发还需要确认的事项

## 1. 当前已经确定的方向

目前已经足够明确的部分：

- 游戏核心感受：原来中国读书是这样子的
- 主角定位：完全玩家投射型
- 开场方式：出发前夜开场，通过回忆进入申请与出发准备
- 主舞台：上海
- 路线：学术、本地融入、国际学生视角、职业发展、城市机会 / 创业都保留
- 数值风格：中等
- 经济压力：中等
- 感情线：有，但不是主轴
- 结局风格：现实与理想混合
- 文风：青年感强
- 关系系统口径：关系系统分为圈层关系 `guanxi` 和个人关系 `relationships`
- 主角申请阶段：本科
- 学校：可使用虚构上海高校
- 专业方向：商科 / 理工 / 人文三类
- 出发前夜场景：整理行李 + 手机消息 + 翻看材料 + 航班确认
- 语言口径：设计文档使用中文，实际游戏内玩家可见文本使用英文

这些已经足够开始写第一阶段内容。

## 3. 仍建议保留为设计前提的事项

这些会直接影响 `序章 + 申请期` 怎么写。

## 3.1 主角国籍是否固定

当前主角是玩家投射型，但还要决定是否需要固定一个出发国家。

建议方案：

- 不固定具体国家
- 用宽泛背景代替国籍
- 继续沿用现有的三类出身：邻近亚洲、西方、全球南方 / 发展中地区

原因：

- 更适合玩家投射
- 不容易把文本写死
- 也能保留现有数值差异

## 3.2 主角申请的学历阶段

已确认主角申请阶段为：

- 本科

设计调整：

- 申请期围绕本科申请展开
- 重点写第一次真正离开原有生活系统
- 家庭期待、专业选择、宿舍生活、同龄人关系应更重要

原因：

- 本科更贴合“第一次完整体验中国读书”的感受
- 更容易写出青年感、同龄人关系和校园生活

## 3.3 主角申请的学校类型

已确认可以使用虚构上海高校。

建议方案：

- 上海综合型重点大学
- 有国际学生宿舍
- 有较强职业资源
- 校园与城市连接紧密

原因：

- 能同时承载校园生活、行政流程、国际学生圈、本地同学圈和职业线

## 3.4 专业方向是否一开始就选

已确认保留专业方向选择。

建议三类：

- 商科 / 经管
- 理工 / 计算机
- 人文 / 语言 / 传播

作用：

- 影响申请材料
- 影响导师线
- 影响课程事件
- 影响职业与结局方向

## 3.5 出发前夜的具体场景

已确认采用组合场景。

建议场景组合：

- 整理行李
- 查看录取通知和签证
- 手机收到家人、朋友、同学消息
- 翻看申请时留下的材料
- 最后确认第二天的航班

作用：

- 自然触发回忆
- 不显得强行倒叙
- 也方便把前期输入信息嵌入剧情

## 3.6 第一版是否使用真实学校名

已确认：

- 用虚构但可信的上海高校

原因：

- 写作自由更大
- 避免现实学校细节不准确
- 后续如果需要，可以再做现实高校版本

## 4. 可以边做边决定的事项

这些不阻塞第一阶段写作。

## 4.1 角色最终数量

可以先确定角色功能，不急着确定最终人数。

第一版建议至少保留：

- 导师 / 学术入口
- 本地同学 / 宿舍入口
- 国际学生朋友
- 职业或创业入口
- 城市生活入口

## 4.2 具体城市支线

第一版主舞台是上海，其他城市可以后面再精细化。

建议先保留：

- 北京：交流 / 学术 / 旅行
- 杭州：科技 / 短途旅行
- 成都：生活节奏 / 文化体验
- 广州：贸易 / 商业
- 西安：历史文化

## 4.3 感情线深度

当前已经确定：

- 有，但不是主轴

后面可以在角色设计阶段决定：

- 哪个角色可发展感情线
- 感情线是否影响结局
- 感情线是否可保持开放结尾

## 4.4 结局数量

不需要现在锁死。

建议第一版先设计：

- 5 个主结局
- 2 到 3 个失败或遗憾结局
- 若干角色尾声
